Project Overview

Developed a machine learning pipeline to predict cinema attendance using historical booking data from two platforms: BookNow (online aggregator) and CinePOS (on-site ticketing system). The goal was to forecast daily audience counts for March 2024 onwards based on 14 months of historical data.

Business Problem

Theaters need accurate attendance forecasts to optimize staffing levels, inventory management, and show scheduling. This project addresses that need by analyzing booking patterns, temporal trends, and theater characteristics.

Feature Engineering Pipeline

Engineered 47 features from raw data to capture temporal dependencies, booking patterns, and theater characteristics.

Feature Categories

  • Lag Features (4): audience_count shifted by 1, 7, 14, 28 days
  • Rolling Statistics (6): Mean & std for 7, 14, 30-day windows
  • Exponential Weighted MA (2): EWMA with spans 7 & 21
  • Trend Indicators (2): 7-14 day trend, 1-7 day momentum
  • Booking Aggregations (9): Daily sum, mean, count by theater
  • Temporal Encoding (5): Month, day_of_week, weekend flag, etc.
  • Categorical (4): Theater type, area, day name (label encoded)
  • Geographic (2): Latitude, longitude (anonymized)

Feature Engineering Code

# Create lag features lags = [1, 7, 14, 28] for lag in lags: train_df[f'lag_{lag}'] = train_df.groupby('book_theater_id')[ 'audience_count' ].shift(lag) # Rolling statistics windows = [7, 14, 30] for window in windows: train_df[f'roll_mean_{window}'] = train_df.groupby( 'book_theater_id' )['audience_count'].shift(1).rolling(window, min_periods=1).mean() train_df[f'roll_std_{window}'] = train_df.groupby( 'book_theater_id' )['audience_count'].shift(1).rolling(window, min_periods=1).std() # Exponential weighted moving average for span in [7, 21]: train_df[f'ewm_{span}'] = train_df.groupby('book_theater_id')[ 'audience_count' ].shift(1).ewm(span=span, adjust=False).mean()

Data Preprocessing Steps

  1. Label Encoding: Converted 4 categorical columns (theater_id, day_of_week, theater_type, theater_area) to numeric using LabelEncoder
  2. Missing Value Imputation: Applied KNN Imputer (k=5, distance-weighted) to handle NaN values in lag/rolling features
  3. Feature Scaling: StandardScaler normalization applied to ensure all features have zero mean and unit variance
  4. Train-Test Split: Temporal split - trained on Jan 2023-Feb 2024, predict March 2024 onwards
